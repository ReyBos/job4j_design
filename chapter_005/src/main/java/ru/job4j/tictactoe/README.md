<h2>Крестики-нолики ООД</h2>
<p>
В этом задании мы будем проектировать классическую игру крестики-нолики.
</p>
<p>
Исходя из описания игры в нашей игре появляются следующие элементы: два игрока, поле, фишки, правила.
<br>Это и будет доменной моделью нашего приложения.
</p>
<p>
Следуя принципу единой ответственности опишем действия выполняемые элементами программы.
<br>Игрок - выполняет ход.
<br>Поле - анализирует процесс игры (игра завершена или может быть продолжена)
<br>Фишка  - занимается отображением (крестик или нолик).
<br>Правила - определяет очередность и возможность ходов.
</p>
<h2>Задание</h2>
<ul>
<li>
Описать доменные модели игры крестики-нолики через Java interface.
Если Вы хотите добавить новые модели в приложение, то в ответе укажите почему они нужны. 
</li>
<li>
Реализуйте классы для консольного варианта.
</li>
<li>
Каждый класс должен тестироваться отдельно.
</li>
</ul>
<h2>Описание моделей</h2>
<ul>
    <li><strong>Point</strong> - хранятся координаты занятой ячейки и игрок сделавший ход</li>
    <li><strong>Player</strong> - делает ход, результат Point</li>
    <li><strong>Mark</strong> - графическое отображение метки игрока</li>
    <li><strong>Store</strong> - хранит ходы игроков</li>
    <li><strong>Rule</strong> - определяет очередность ходов</li>
    <li><strong>Field</strong> - анализирует процесс игры</li>
</ul>
